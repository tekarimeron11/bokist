# 解説テンプレート（v2 草案 — 知識ゼロ前提）

> 対象: 日商簿記3級 第1問 仕訳問題（`content/journals.json`）
> **想定読者**: 簿記の知識ゼロの先輩（20代女性）。テキストはまだ開いていない。借方・貸方・資産・負債・三分法 などの言葉も知らない前提
> 利用シーン: スマホ・電車内で、問題を間違えた直後に読む
> 体裁: デスマス調・温かみあり・上から目線にならない

## 共通フォーマット

各テンプレートは下記の **A〜E の5セクション** で構成します。

| セクション | 内容 | 役割 |
|---|---|---|
| A. ストーリー | 1〜2文で日常語に翻訳した「何が起きたか」 | 取引のイメージ化 |
| B. 用語の整理 | このテンプレで使う用語を最低限の3〜5個、1〜2文で定義 | 知識の前提埋め |
| C. 仕訳のルール | 「資産が増えたら借方／減ったら貸方」など、このテンプレで使う行だけの早見表 | 仕訳の機械的ルール提示 |
| D. 構造化解説（4要素） | `essence` / `debitWhy` / `creditWhy` / `takeaway` の4ブロック | 本問の説明 |
| E. ポイントの一文 | takeaway と同じ、≦30字の覚える一言 | 暗記用フック |

D の構造化解説は `Question.structured: StructuredExplanation` として保存する想定です（REQUIREMENTS.md §7.1 / §8.5 と整合）。

```ts
type StructuredExplanation = {
  essence:   string;  // 取引の本質（1文）
  debitWhy:  string;  // 借方の理由
  creditWhy: string;  // 貸方の理由
  takeaway:  string;  // ポイント（≦30字）
};
```

**ルール**:
1. 各テンプレ内で、**B で定義していない用語を D 以降で使ってはいけません**。
2. 同じ用語を複数回使う場合、最初の登場時に `<Term>` タグを付けます（用語ポップアップ参照、§8.5）。
3. 数字・問題文は既存の `content/journals.json` と完全に一致させます（差し替え時の1対1マッピングのため）。

## テンプレート 1: 三分法による仕入（諸掛りなし）

- **論点名 / topicId**: 三分法による仕入 / `shouhin-sanbunpou-shiire`
- **章**: shouhin
- **典型例の問題文**:
  > A商店から商品¥80,000を仕入れ、代金は掛けとした。
- **正解仕訳**:

| 借方 | 金額 | 貸方 | 金額 |
|---|---|---|---|
| `purchase`（仕入） | 80,000 | `accounts_payable`（買掛金） | 80,000 |

### A. ストーリー

先輩がカフェ開業の準備で、A商店からコーヒー豆を ¥80,000 分まとめて買いました。代金はその場で払わず、月末にまとめて払う約束（いわゆる「ツケ」）にしてもらった、という取引です。

### B. 用語の整理

- **<Term>借方</Term>（かりかた）**: 仕訳を書く紙の **左側** に書く欄のことです。日常語の「借りる」とは関係なく、ただの位置の名前と思ってください。
- **<Term>貸方</Term>（かしかた）**: 仕訳を書く紙の **右側** に書く欄のことです。これも「貸す」という意味とは関係ありません。
- **5分類**: 簿記ではお金や物の動きを「資産・負債・純資産・収益・費用」の5種類に分けて記録します。種類によって「増えたとき左に書く／右に書く」のルールが決まっています。
  - **資産**: 会社が持っている物・お金・あとでもらえる権利（例: 現金、商品、売掛金）
  - **負債**: 会社が払う義務・あとで返すもの（例: 買掛金、借入金）
  - **費用**: 商売のために使ったお金・コスト（例: 仕入、給料、家賃）
- **<Term>三分法</Term>**: 商品の売り買いを「仕入」「売上」「繰越商品」の3つの勘定で記録するルールです。3級ではこのルールを使います。
- **仕入**（しいれ）: 商品を買い入れたときに使う、**費用**の勘定です。三分法では商品を仕入れた金額をここに集めます。
- **<Term>買掛金</Term>**（かいかけきん）: 「商品を仕入れたあと、代金を払う約束」を表す **負債** の勘定です。読み方は「かいかけきん」。

### C. 仕訳のルール（このテンプレで使う分）

```
費用が増えたら → 借方（左）
負債が増えたら → 貸方（右）
```

このテンプレでは「商品を買って、ツケが発生した」だけなので、上の2行で足ります。

### D. 構造化解説

- **essence（取引の本質）**:
  商品 ¥80,000 を受け取った代わりに、後で代金を払う約束（ツケ）が ¥80,000 ぶん発生した取引です。
- **debitWhy（借方の理由）**:
  商品の仕入は **費用** の発生です。上のルール表のとおり、費用が増えたら借方（左）に書くので、左に「仕入 80,000」と書きます。
- **creditWhy（貸方の理由）**:
  代金を後で払うツケ（買掛金）は **負債** の発生です。負債が増えたら貸方（右）なので、右に「買掛金 80,000」と書きます。
- **takeaway（ポイント）**:
  仕入＝費用、ツケで買えば買掛金。

### E. ポイントの一文

> 仕入は費用（左）、ツケは買掛金（右）。

## テンプレート 2: 諸掛り込み（仕入諸掛り）

- **論点名 / topicId**: 三分法による仕入（仕入諸掛り） / `shouhin-sanbunpou-shiire`
- **章**: shouhin
- **典型例の問題文**:
  > B商店から商品¥50,000を仕入れ、引取運賃¥2,000を現金で支払った。商品代金は掛けとした。
- **正解仕訳**:

| 借方 | 金額 | 貸方 | 金額 |
|---|---|---|---|
| `purchase`（仕入） | 52,000 | `accounts_payable`（買掛金） | 50,000 |
|  |  | `cash`（現金） | 2,000 |

### A. ストーリー

先輩は B商店から ¥50,000 ぶんのコーヒー豆を仕入れました。お店までの運賃が ¥2,000 かかり、これは現金でその場で払いました。商品本体の ¥50,000 はツケにしてもらった、という取引です。お店に商品を届けるまでに合計 ¥52,000 のお金がかかった、と覚えてください。

### B. 用語の整理

- **借方 / 貸方**: 仕訳の左側／右側のことです（テンプレ1参照）。
- **資産**: 会社が持っているお金・物・もらえる権利。**現金** はその代表です。
- **負債**: 会社が払う義務。**買掛金**（仕入のツケ）はここに入ります。
- **費用**: 商売のために使ったお金。**仕入** はここに入ります。
- **<Term>諸掛り</Term>**（しょがかり）: 商品を売ったり買ったりするときに本体価格とは別にかかる費用（運賃・手数料など）のことです。
- **<Term>仕入諸掛り</Term>**: 商品を **仕入れる** ときにかかった諸掛り（運賃など）のこと。簿記では原則として **「仕入」勘定に上乗せして記録する** ルールです。

### C. 仕訳のルール（このテンプレで使う分）

```
費用が増えたら → 借方（左）
負債が増えたら → 貸方（右）
資産が減ったら → 貸方（右）   ← 現金で払うと現金が減るのでここに登場
```

### D. 構造化解説

- **essence（取引の本質）**:
  商品本体 ¥50,000 と運賃 ¥2,000 がセットで発生し、商品を手元に届けるまでのコストが合計 ¥52,000 になった取引です。
- **debitWhy（借方の理由）**:
  仕入諸掛り（運賃 ¥2,000）は商品本体に上乗せして仕入勘定に含める決まりなので、仕入は ¥50,000 + ¥2,000 = **¥52,000** で借方（左）に書きます。
- **creditWhy（貸方の理由）**:
  代金 ¥50,000 はツケなので **買掛金（負債の増加）**、運賃 ¥2,000 はその場で払ったので **現金（資産の減少）**。どちらも貸方（右）側に分けて書きます。
- **takeaway（ポイント）**:
  運賃も仕入の一部、合算してから記録。

### E. ポイントの一文

> 運賃は仕入に足してから書く。

## テンプレート 3: 売上戻り（返品）

- **論点名 / topicId**: 返品・値引（売上戻り） / `shouhin-henpin`
- **章**: shouhin
- **典型例の問題文**:
  > 先日C商店に掛けで販売した商品のうち¥15,000が品違いにより返品された。
- **正解仕訳**:

| 借方 | 金額 | 貸方 | 金額 |
|---|---|---|---|
| `sales`（売上） | 15,000 | `accounts_receivable`（売掛金） | 15,000 |

### A. ストーリー

先輩はこの前 C商店に商品をツケで売っていたのですが、そのうち ¥15,000 ぶんが「頼んだのと違う品物だった」と戻ってきました。売った話と、後でもらえる予定だったお金を、その分だけ「なかったこと」にする取引です。

### B. 用語の整理

- **借方 / 貸方**: 仕訳の左側／右側のことです（テンプレ1参照）。
- **収益**: 5分類のひとつ。商売で稼いだお金（売上、受取利息など）。
- **資産**: 持っている物・お金・もらえる権利。
- **売上**（うりあげ）: 商品を売ったときに使う、**収益** の勘定です。
- **<Term>売掛金</Term>**（うりかけきん）: 「商品を売ったので、代金を後でもらえる」という権利を表す **資産** の勘定です。読み方は「うりかけきん」。
- **売上戻り**（うりあげもどり）: 一度売った商品が、品違いや不良で戻ってくることです。仕入側からは「仕入戻し」と呼びます。
- **<Term>振替</Term>** / 逆仕訳: いったん書いた仕訳を取り消すために、借方と貸方を入れ替えてもう一度書く操作のことです。返品は典型的な逆仕訳のシーンです。

### C. 仕訳のルール（このテンプレで使う分）

```
収益が減ったら → 借方（左）   ← 売上を取り消す
資産が減ったら → 貸方（右）   ← 売掛金を取り消す
```

通常は「収益が **増えたら** 貸方」「資産が **増えたら** 借方」が基本ですが、返品では **逆向き** になります。

### D. 構造化解説

- **essence（取引の本質）**:
  一度売った商品 ¥15,000 が戻ってきたので、過去に立てた「売上」と「あとでもらう代金」を、その分だけ取り消す取引です。
- **debitWhy（借方の理由）**:
  最初に貸方（右）に書いた売上を ¥15,000 ぶん打ち消すため、今回は逆側の **借方（左）に売上 15,000** を書きます。収益が減るときは借方、と覚えてください。
- **creditWhy（貸方の理由）**:
  代金をもらえる権利だった売掛金（資産）も同額消えます。資産が減ったら貸方なので、右に **売掛金 15,000** を書きます。
- **takeaway（ポイント）**:
  返品は元の仕訳をそのまま逆向きに。

### E. ポイントの一文

> 売上戻り＝最初の仕訳を裏返す。

## テンプレート 4: 売掛金の決済（小切手で回収）

- **論点名 / topicId**: 売掛・買掛の決済 / `urikake-kessai`
- **章**: urikake
- **典型例の問題文**:
  > F商店に対する売掛金¥100,000が、同店振出しの小切手で回収された。
- **正解仕訳**:

| 借方 | 金額 | 貸方 | 金額 |
|---|---|---|---|
| `cash`（現金） | 100,000 | `accounts_receivable`（売掛金） | 100,000 |

### A. ストーリー

先輩は前に F商店へ商品を売って、代金 ¥100,000 を「あとでもらう」約束にしていました。今日その代金が、F商店が書いた **小切手** という紙で支払われました。「もらえる予定」だった ¥100,000 が、実際に手元のお金に変わった取引です。

### B. 用語の整理

- **借方 / 貸方**: 仕訳の左側／右側のことです（テンプレ1参照）。
- **資産**: 会社が持っているお金・物・もらえる権利。
- **現金**（簿記での意味）: 紙幣・硬貨だけでなく、銀行ですぐ現金化できる **小切手** や送金小切手なども簿記では「現金」勘定で扱います。普段の感覚より少し範囲が広いです。
- **小切手**（こぎって）: 「この金額を銀行から払ってください」と書いた紙のこと。受け取った人は銀行に持っていけば現金がもらえます。
- **<Term>振出</Term>**（ふりだし）: 小切手や手形を発行して相手に渡すこと。「F商店振出しの小切手」＝F商店が発行した小切手、という意味です。
- **売掛金**: 商品を売ったあとで「代金をもらえる権利」を表す **資産** の勘定（テンプレ3参照）。
- **他人振出小切手**: 取引相手（自分の会社以外）が振り出した小切手のこと。受け取った瞬間に簿記では「現金」として扱います。

### C. 仕訳のルール（このテンプレで使う分）

```
資産が増えたら → 借方（左）   ← 現金が増える
資産が減ったら → 貸方（右）   ← 売掛金が消える
```

両方とも資産の動きですが、一方が増えて一方が減る「資産同士の振り替え」になっています。

### D. 構造化解説

- **essence（取引の本質）**:
  「あとで ¥100,000 もらえるはずだった権利」が、実際の小切手という現金に変わった取引です。資産の中身が入れ替わっただけで、資産の合計は変わりません。
- **debitWhy（借方の理由）**:
  他人が振り出した小切手は簿記では「現金」勘定で扱う決まりです。資産（現金）が ¥100,000 増えたので、借方（左）に **現金 100,000** と書きます。
- **creditWhy（貸方の理由）**:
  もらえる権利だった売掛金（資産）が ¥100,000 ぶん消えました。資産が減ったら貸方なので、右に **売掛金 100,000** を書きます。
- **takeaway（ポイント）**:
  他人の小切手＝現金あつかい。

### E. ポイントの一文

> もらった小切手は現金として書く。

## テンプレート 5: 貸倒引当金の設定（差額補充法）

- **論点名 / topicId**: 貸倒引当金の設定 / `urikake-hikiate-settei`
- **章**: urikake
- **典型例の問題文**:
  > 決算につき、売掛金期末残高¥500,000に対して2%の貸倒引当金を差額補充法により設定する。貸倒引当金の残高は¥3,000である。
- **正解仕訳**:

| 借方 | 金額 | 貸方 | 金額 |
|---|---|---|---|
| `doubtful_provision`（貸倒引当金繰入） | 7,000 | `allowance_doubtful`（貸倒引当金） | 7,000 |

### A. ストーリー

先輩のお店には売掛金（あとでお金をもらえる約束）が ¥500,000 ぶんあります。でも、お客さんが倒産したりして 100% 全部回収できるとは限りません。「全体の 2% くらいは回収できないかも」と見積もって、その分のお金をあらかじめ「予備費」として帳簿に積んでおきます。すでに ¥3,000 積んであるので、今回は足りない分の ¥7,000 だけ追加する、という取引です。

### B. 用語の整理

- **<Term>決算整理</Term>**（けっさんせいり）: 1年に1回、帳簿を正しい姿に整える作業のことです。減価償却・貸倒引当金・経過勘定などの仕訳がここで登場します。
- **貸倒れ**（かしだおれ）: 売掛金などが回収できなくなることです。
- **<Term>貸倒引当金</Term>**（かしだおれひきあてきん）: 将来の貸倒れに備えてあらかじめ「予備費」として準備しておく金額。売掛金から差し引いて表示する、ちょっと特別な勘定です。
- **<Term>差額補充法</Term>**（さがくほじゅうほう）: 必要な引当金額（今回は ¥10,000）と、すでにある残高（¥3,000）との **差額** だけを追加する方法のことです。3級ではこの方法を使います。
- **貸倒引当金繰入**（かしだおれひきあてきんくりいれ）: 今期に追加で積み増した金額を **費用** として記録するための勘定です。
- **評価勘定**（ひょうかかんじょう）: ある資産（売掛金など）の金額を補正（差し引き）するための勘定。貸倒引当金がこのタイプ。「資産のマイナス側」だと思ってください。

### C. 仕訳のルール（このテンプレで使う分）

```
費用が増えたら         → 借方（左）   ← 貸倒引当金繰入が増える
評価勘定（資産マイナス）が増えたら → 貸方（右）   ← 貸倒引当金が増える
```

「資産のマイナス側が増える」というのは、見た目には貸方（右）に書きます。覚えるなら「貸倒引当金は **必ず貸方側に積み上がる**」と機械的に覚えるのが早いです。

### D. 構造化解説

- **essence（取引の本質）**:
  売掛金 ¥500,000 のうち 2%（¥10,000）が回収不能になるかも、と見積もっておく仕訳です。すでに ¥3,000 あるので、不足分の ¥7,000 だけ追加で積みます。
- **debitWhy（借方の理由）**:
  追加で積む ¥7,000 は **当期の費用** として扱う決まりです（貸倒引当金繰入）。費用が増えたら借方（左）なので、左に **貸倒引当金繰入 7,000** を書きます。
- **creditWhy（貸方の理由）**:
  予備費そのもの（貸倒引当金）が ¥7,000 増えるので、貸方（右）に **貸倒引当金 7,000** を書きます。これは売掛金から後で差し引く「マイナス資産」のような位置づけの勘定です。
- **takeaway（ポイント）**:
  必要額 − 残高 ＝ 今回繰り入れる金額。

### E. ポイントの一文

> 不足分だけ繰り入れて足し算で補う。

## テンプレート 6: 減価償却（定額法・間接法）

- **論点名 / topicId**: 減価償却（定額法） / `kotei-genkashoukyaku`
- **章**: kotei
- **典型例の問題文**:
  > 決算につき、期首に取得した備品（取得原価¥300,000、残存価額¥0、耐用年数5年）の減価償却を定額法・間接法により行う。
- **正解仕訳**:

| 借方 | 金額 | 貸方 | 金額 |
|---|---|---|---|
| `depreciation`（減価償却費） | 60,000 | `accumulated_dep`（減価償却累計額） | 60,000 |

### A. ストーリー

先輩のお店で ¥300,000 のエスプレッソマシン（備品）を期の最初に買いました。これは何年も使えるものですが、毎年少しずつ古くなって価値が下がります。「5年使い切る予定」なら、1年あたり ¥60,000 ずつ価値が減ったことにします。今期分の ¥60,000 を **費用** として記録する仕訳です。

### B. 用語の整理

- **決算整理**: 1年に1回、帳簿を正しく整える作業（テンプレ5参照）。
- **取得原価**（しゅとくげんか）: 物を手に入れるのに払った金額の合計（本体価格＋付随費用）。今回は ¥300,000。
- **残存価額**（ざんぞんかがく）: 耐用年数を使い切ったあとに残ると見積もった価値。今回は ¥0。
- **耐用年数**（たいようねんすう）: 「だいたい何年使える物か」という見積もりの年数。今回は 5年。
- **<Term>減価償却</Term>**（げんかしょうきゃく）: 長く使う物の価値が毎年少しずつ減っていく分を、毎期 **費用** として記録する手続きです。
- **定額法**（ていがくほう）: 毎年同じ金額ずつ価値が減るとして計算する方法。**(取得原価 − 残存価額) ÷ 耐用年数 ＝ 1年分**。
- **<Term>間接法</Term>**（かんせつほう）: 減価償却で、備品そのものの金額には触らず、別の **「減価償却累計額」** という勘定にその年の減少分を積み上げていく方法。3級ではこの方法を使います。
- **減価償却累計額**: これまで償却してきた金額の累計。備品の金額から後で差し引いて表示する評価勘定（資産のマイナス側）。
- **減価償却費**: その年に減った価値を **費用** として記録するための勘定。

### C. 仕訳のルール（このテンプレで使う分）

```
費用が増えたら          → 借方（左）   ← 減価償却費が増える
評価勘定（資産マイナス）が増えたら → 貸方（右）   ← 減価償却累計額が増える
```

形は貸倒引当金（テンプレ5）とそっくりです。「費用の発生」と「資産マイナス側の増加」のペア、と覚えてください。

### D. 構造化解説

- **essence（取引の本質）**:
  ¥300,000 の備品を5年で使い切る前提で、今期に減ったぶんの価値 ¥60,000 を費用として記録する仕訳です。
- **debitWhy（借方の理由）**:
  計算は (¥300,000 − ¥0) ÷ 5年 ＝ **¥60,000**。これを当期の費用「減価償却費」として借方（左）に書きます。費用が増えたら借方、のルールどおりです。
- **creditWhy（貸方の理由）**:
  間接法では備品そのもの（¥300,000）には手を付けず、減った分を「減価償却累計額」という別勘定に積み上げます。資産のマイナス側が ¥60,000 増えるので、貸方（右）に **減価償却累計額 60,000** を書きます。
- **takeaway（ポイント）**:
  (取得原価 − 残存価額) ÷ 耐用年数 ＝ 1年分。

### E. ポイントの一文

> 1年分を費用に、累計額は右に積む。

## テンプレート 7: 経過勘定 — 費用の見越し（未払給料）

- **論点名 / topicId**: 見越し（未払・未収） / `keika-mikoshi`
- **章**: keika
- **典型例の問題文**:
  > 決算（3月31日）につき、給料は月末締め翌月10日払いのため、3月分給料¥200,000が未払いとなっている。これを見越計上する。
- **正解仕訳**:

| 借方 | 金額 | 貸方 | 金額 |
|---|---|---|---|
| `salary`（給料） | 200,000 | `accrued_salary`（未払給料） | 200,000 |

### A. ストーリー

先輩のお店では、給料を「月末で締めて、翌月10日に払う」ルールにしています。3月31日（決算日）時点で、3月分の給料 ¥200,000 はもう「働いてもらった分」として発生しているのに、支払いは4月10日。お金はまだ動いていないけれど、3月の費用として記録しておかないと帳簿がウソになります。今期の費用として **先に立てておく** のがこの取引です。

### B. 用語の整理

- **決算整理**: 1年に1回、帳簿を正しく整える作業（テンプレ5参照）。
- **<Term>経過勘定</Term>**（けいかかんじょう）: 期末時点で「当期分」と「翌期分」が混ざっている費用や収益を、正しい期に振り分けるための調整勘定の総称。前払・前受・未払・未収の4種類があります。
- **<Term>見越し</Term>**（みこし）: 当期に発生したのに、まだ支払い・受取りが済んでいない分を当期に **先に計上する** 処理。
- **発生主義**（はっせいしゅぎ）: お金の出入りではなく「発生した事実」に合わせて費用や収益を記録する考え方。簿記の基本ルールです。
- **給料**（簿記の勘定として）: 従業員に払う給料を **費用** として記録する勘定。
- **未払給料**（みばらいきゅうりょう）: 当期に発生した給料のうち、まだ支払っていない金額。「これから払う義務」なので **負債** の勘定です。

### C. 仕訳のルール（このテンプレで使う分）

```
費用が増えたら → 借方（左）   ← 給料が発生
負債が増えたら → 貸方（右）   ← 未払給料という払う義務が発生
```

### D. 構造化解説

- **essence（取引の本質）**:
  3月分の給料 ¥200,000 はもう発生しています。支払いは来月でも、費用としては今期に記録するための仕訳です。
- **debitWhy（借方の理由）**:
  当期に発生した給料は当期の費用です。お金をまだ払っていなくても、費用「給料」として借方（左）に立てます（発生主義）。
- **creditWhy（貸方の理由）**:
  まだ払っていない ¥200,000 は「これから払う義務」なので、**未払給料**（負債）として貸方（右）に書きます。負債が増えたら貸方、のルールどおりです。
- **takeaway（ポイント）**:
  発生したら払う前でも費用に。

### E. ポイントの一文

> 払ってなくても、発生した費用は今期に。

## テンプレート 8: 訂正仕訳（金額誤り）

- **論点名 / topicId**: 訂正仕訳 / `sonota-teisei`
- **章**: sonota
- **典型例の問題文**:
  > 消耗品¥5,000を購入して現金で支払った際、誤って消耗品費¥500/現金¥500と仕訳していたことが判明した。これを訂正する。
- **正解仕訳**:

| 借方 | 金額 | 貸方 | 金額 |
|---|---|---|---|
| `supplies_fee`（消耗品費） | 4,500 | `cash`（現金） | 4,500 |

### A. ストーリー

先輩は前にコピー用紙やペンなど ¥5,000 ぶんの事務用品を現金で買いましたが、帳簿に書いたときに桁を間違えて「¥500 で買ったこと」にしてしまいました。今、その間違いに気づいたので、正しい金額にするために **不足している ¥4,500 を後から追加で書き足す** 取引です。

### B. 用語の整理

- **借方 / 貸方**: 仕訳の左側／右側のことです（テンプレ1参照）。
- **費用**: 商売のために使ったお金。
- **資産**: 会社が持っているお金・物。**現金** はその代表。
- **消耗品費**（しょうもうひんひ）: 文房具・コピー用紙など、すぐ使い切るものの代金を記録する **費用** の勘定。
- **現金**（簿記の勘定として）: 紙幣・硬貨・他人振出小切手など、すぐお金として使えるもの。**資産**。
- **<Term>訂正仕訳</Term>**（ていせいしわけ）: 過去にした仕訳の誤りを直すための仕訳。基本の考え方は **「誤りを打ち消す逆仕訳」＋「正しい仕訳」を合計する** こと。同じ向きに動かすなら **差額だけを書けば OK** です。

### C. 仕訳のルール（このテンプレで使う分）

```
費用が増えたら → 借方（左）   ← 消耗品費を ¥4,500 追加で増やす
資産が減ったら → 貸方（右）   ← 現金を ¥4,500 追加で減らす
```

### D. 構造化解説

- **essence（取引の本質）**:
  本当は ¥5,000 の支払いだったのに ¥500 だけしか帳簿に書いていなかったので、不足分 ¥4,500 を後から書き足して帳尻を合わせる仕訳です。
- **debitWhy（借方の理由）**:
  消耗品費は本来 ¥5,000 で計上すべきところ ¥500 しか乗っていません。不足分の ¥4,500 を借方（左）に追加で書いて、費用を正しい金額まで増やします。
- **creditWhy（貸方の理由）**:
  支払った現金も本来は ¥5,000 ぶん減っているはず。¥500 ぶんしか減らしていなかったので、不足分 ¥4,500 を貸方（右）に追加して現金の減少を正します。
- **takeaway（ポイント）**:
  誤りを取り消す仕訳＋正しい仕訳＝差額だけを書く。

### E. ポイントの一文

> 訂正仕訳は「正しい状態 − 今の状態」だけ書く。

## 補足: テンプレ運用上のメモ

1. **長さの目安**: A〜E 全体で **600〜1000 字** を目安。今回 v2 草案は概ね 800〜1000 字に収まっています（テンプレ1, 5, 6 はやや長め）。
2. **金額・問題文**: `content/journals.json` 既存の値を改変せず転載。下流の batch-apply ワーカーが1対1で差し替えできる形を維持。
3. **`<Term>` の付与ルール**: 同一テンプレ内で同じ用語が複数回登場する場合、**最初の1回のみ** タグ化（§7.1 / §8.5 と整合）。
4. **B セクションの書き順**: 「最も基礎的な概念から、その上に乗る応用まで」の順。例: テンプレ1 では `借方 → 貸方 → 5分類 → 三分法 → 仕入 → 買掛金` の順で積み上げています。
5. **C セクションのミニ早見表**: 該当する行のみを表示。「全部のルールを毎回出さない」設計です。読者が C を見ながら D を読めるよう、UI 上は C と D を同一スクロール画面に置くことを想定しています。
6. **`structured` フィールド適用範囲**: D の4要素（essence / debitWhy / creditWhy / takeaway）を `Question.structured` に格納。A/B/C/E は別途、レンダリング時に「テンプレID」経由で動的に展開する想定（実装はフェーズ1.5 / M4 推奨）。M2 は `structured` のみで開始しても OK。
